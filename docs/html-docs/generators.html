Automatic Bitmap Generators.

In order to describe the cross sections of the transmission lines, you can either use a graphics programme, or one of several programmes which will be written in order to generate bitmaps more quickly. Currently several automatica bitmap generators have been written. Unix manual pages are avialable for all the programmes:
  1. rect_in_rect for a rectangular conductor placed anywhere inside another rectangular conductor.
  2. rect_cen_in_rect for a rectangular conductor placed centrally inside another rectangular conductor.
  3. circ_in_circ for a circular conductor placed inside another circular conductor
  4. circ_in_rect for a circular conductor placed inside a rectangular one
  5. rect_in_circ for a rectangular conductor placed inside a circular conductor. rect_in_circ is currently still under development
  6. sym_strip for a thin conductor equally spaced between two wide groundplanes
Slightly more comprehensive documemtation exists for rect_in_rect The programmes such as rect_in_rect will by default produce a bitmap of around 250 kb, which should be analysed in about minutes at most, depending on the speed of your computer. You can increase or decrease the size of the bitmap, by use of the -b option.

The output from rect_in_rect is sent to standard output. Since this is binary data, it will possibly screw up your terminal unless it is redirectected to a file. In each case, you can either use the > sign and put directly into a file, or use the -o option.
