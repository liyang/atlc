To Do list..

Here is a list of the ways I would like to expand atlc, in a rough order of time.
  1. Implement a better way of enabling a user to change the contrast/brightness in an image. Current, this can only be done with the '-f option to atlc, but this requires a re-run of atlc to do this, which is hardly very useful, given atlc is so CPU intensive. There are faster ways of doing this, just from the binary files (example.E.bin, example.Ex.bin and example.Ey.bin)..

  2. Try other methods of thread syncronisation in the version of the programme that supports multiple CPUs. Currently, threads are created and destroyed to given syncronisation. I think there may be a faster method by the use of 'mutex variables', but this has not be tried.

    3. Usage: rect_in_rect  W H a b c w h Er1 Er2 accuracy_level filename

where W, H, a, b, c, w and h  are all in mm or all in
inches - they *must* use the same units!
Accuracy should be 1, 2, 3, 4, ... 10  which  sets the internal array size.
3 gives is pretty good accuracy - any more than 5 will take ages.

  3. Add support for coupled transmission lines, so permitting computation of the properties of directional couplers. I've tried this before without much success, but I see no theoretical reason that this can not be done. The programme should be able to calculate the odd mode (Zodd) and even mode Zeven impedances of a pair of coupled transmission lines. From those, it should be possible to calculate the properties of a directional coupler, knowing these two impedances and the length over which the lines are coupled. It sounds easy, but in practice I gave up with this once, due to lack of success.

  4. I'm considering adding support for Sun's HPC, which allows many machines on a network to act in parrallel to solve a problem. This is perhaps not needed on atlc, since while it is cpu intensive, it is not massively so. atlc already supports multiple CPUs, but supporting a network of CPUs is probably a bit over the top. However, I'm interested in such concepts, so might do it anyway. I once developed my own system for using a network of 24 computers for modelling light transport in human tissue. However, HPC is far more sophisticated than what I did.
If anyone has any suggestions for how to improve atlc, please email me.
