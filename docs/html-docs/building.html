Building atlc from sources

atlc is highly portable program and should run on any UNIX system and some non-UNIX systems.

An early version for atlc forms part of the NetBSD project and has been ported to 23 different systems, including the Acorn 32, Amiga, Arm32, Atari, Cobalt, Dec Alpha, HP 300, i386 PC, Mac 68000, Power PC, Sun SPARC, VAX etc. atlc has even been run on a Sony Playstation 2!!. In addition to the systems running NetBSD UNIX, atlc has been run on the following operating systems
  1. Solaris 2.5, 8 and Solaris 9 for SPARC
  2. Solaris 8 for Intel
  3. Linux 2.4.17 for a Dec Alpha
  4. Tru64 UNIX 5.1B on a Dec Alpha 600a Personal Workstation
  5. Redhat 7.2 Linux on PC
  6. Redhat 6.2 on SPARC. Fails many tests if configured --with-threads.
  7. Debian 3.0 Linux on SPARC
  8. NetBSD 1.6 on SPARC
  9. OpenBSD
  10. AIX 5.2 on an IBM RS/6000.
  11. IRIX 6.5.16 on an SGI Octane
  12. x86 PC's run Windoze 98 and NT
To install altc you should download the souce code from atlc's download area on SourceForge. It is preferable to also have following software too, although none, apart from a compatible version of make, are essential.
  1. The GNU scientific library (gsl). While not essentail, this is highly reccomended.
  2. GNU make, sometimes installed as gmake. Sun's version of make in /usr/ccs/bin/make is not suitable, although I don't know about other versions of make. The GNU one works fine.
  3. If you have a system with more than one CPU, atlc can be configured to use any number of CPUs in parallel to speed execution, using a POSIX compatable thread library, or pthreads. Sun's Solaris implementation works fine, but any other POSIX library should work. If you have multiple CPUs and don't have a thread library, install GNU Portable Threads,. If your computer has only one CPU, there is no point in building atlc to use multiple threads, as it will slow the program down.
  4. If you have access to a distributed set of computers and want to put a lot of effort in, you might like to help develop an MPI version of atlc. In which case, install the MPICH library. This will enable some tests to check the MPI implementation, but will not do anything useful to atlc now. Hence for 99.999% of users, installing the MPICH library would be a waste of time.

Extract the compressed file, using gunzip, or gzip -d. 
% gzip -d atlc-X.Y.tar.gz
% tar xvf atlc-X.Y.tar
% cd atlc-X.Y
% ./configure
% make
% make check   // This is very important, as it checks the binaries.
If you have multiple CPUs, add the option --with-threads to configure before running make.

A total of 89 self-tests are performed in the make check stage. No tests should fail but several will be skipped. Four tests make use of the t gsl library and so will be skipped unless that is present. Two tests are used to check the operation of the POSIX threads, so will be skipped unless configured with --with-threads. One test checks the operation of the MPICH library, so will be skipped unless configure --with-mpi is run.

Note, you can not run
% configure --with-mpi --with-threads
as these options are mutually exclusive. Because of this, at leasat one test will always be skipped.

Finally you can install the software if you wish. 

% su
# make install.
The make install stage will install:
  1. The executable programs (by default to /usr/local/bin)
  2. The man pages (by default to /usr/local/man/man1)
  3. Some HTML formatted documentation (generally similar to that on the atlc website. By default these are installed to /usr/local/share/atlc/docs/html-docs.
  4. A copy of a paper published on atlc back in the December 1996 issue of QEX. That along with the C source for this very early version is installed to /usr/local/share/atlc/docs/qex_december_1996
  5. Some example files (by default to /usr/local/share/atlc/examples)

To build atlc with support for distributed processing using the MPICH library, run: 
% configure --with-mpi=/path_to_your_mpi_library
There are several other options to the configure script, which can be obtained by running configure --help
