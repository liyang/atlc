Building atlc from sources

atlc is highly portable program and should run on any UNIX system and some non-UNIX systems.

An early version for atlc forms part of the NetBSD project and has been ported to 23 different systems, including the Acorn 32, Amiga, Arm32, Atari, Cobalt, Dec Alpha, HP 300, i386 PC, Mac 68000, Power PC, Sun SPARC, VAX etc. atlc has even been run on a Sony Playstation 2!!. In addition to the systems running NetBSD UNIX, atlc has been run on the following platforms.
  1. Dual processor Sun Ultra 60 running Solaris 8l
  2. Quad processor (4x450 MHz) Sun Ultra 80 running Solaris 9
  3. 64-bit Dec Alpha, running Linux 2.4.17
  4. x86 PC running linux 2.4.13
  5. Sun (unknown) running Linux 2.2.18pre21
  6. Dual processor DEC Alpha running Linux 2.4.17
  7. Sun R220 running Solaris 8
  8. x86 PC's running Windows 98 and NT
To install altc you should download the souce code from atlc's download area on SourceForge. It is preferable to also have following software too, although none, apart from a compatible version of make, are essential.
  1. The GNU scientific library (gsl). While not essentail, this is highly reccomended.
  2. GNU make, sometimes installed as gmake. Sun's version of make in /usr/ccs/bin/make is not suitable, although I don't know about other versions of make. The GNU one works fine.
  3. If you have a system with more than one CPU, atlc can be configured to use any number of CPUs in parallel, using a POSIX compatable thread library, or pthreads. Sun's Solaris implementation works fine, but any other POSIX library should work. If you don't have one you should install GNU Portable Threads,. If your computer has only one CPU, there is no point in building atlc to use multiple threads, as it will slow the program down.
  4. If you have access to a network of computers and wish to use some highly experimental (i.e. probably unstable) distributed processing code, you will need to have installed the MPICH library, which is a portable implementation of the Message Passing Interface standard. Using the MPICH library a network of computers can be used to solve problems more quickly. The author of atlc , Dr. David Kirkby, has not seen this work, but has been told it works under GNU/Linux. In contrast, using multiple processors on one computer via the use of threads works well.

Extract the compressed file, using gunzip, or gzip -d. 
% gzip -d atlc-X.Y.tar.gz
% tar xvf atlc-X.Y.tar
% cd atlc-X.Y
% ./configure
% make 
% make check   // This is very important, as it checks the binaries.
This should run around 90 self-tests. All should pass, but if you don't have the GNU scientific library.(gsl), approximately 4 tests will fail. Without gsl, atlc suffers some loss of functionality, so these tests fail. 
% su
# make install.
The make install stage will install:
  1. The executable programs (by default to /usr/local/bin)
  2. The man pages (by default to /usr/local/man/man1)
  3. Some HTML formatted documentation (generally similar to that on the atlc website. By default these are installed to /usr/local/share/atlc/docs/html-docs.
  4. A copy of a paper published on atlc back in the December 1996 issue of QEX. That along with the C source for this very early version is installed to /usr/local/share/atlc/docs/qex_december_1996
  5. Some example files (by default to /usr/local/share/atlc/examples)

If you have a system with more than one CPU and wish to exploit multiple-processing using threads, you should then (after first building a single threaded version above), re-run the configure script. 
% configure --with-threads
% make 
% su
# make install.
To build atlc with support for distributed processing using the MPICH library, run: 
% configure --with-mpi=/path_to_your_mpi_library
There are several other options to the configure script, which can be obtained by running configure --help

Note, atlc should first be configured without threads and make check run. The algorithm used for a threaded version of atlc is slightly different, so will result in a lot of tests failinig, as the files are slightly different and so have different checksums.It is hoped to fixed this problem soon, so that the tests can be run properly on a threaded version of atlc.
