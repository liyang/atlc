Multi-processing with atlc

The executable program atlc, which is part of the atlc package of programs, can be quite CPU intensive. This is especially troublesome if atlc is executed a large number of times in an itterative loop, as the program desgin_coupler will do if asked to find the phyical dimensions of a directional coupler.

To reduced execution time, the program atlc can be configured to support multiple processors in one of two very diferent ways.
  1. Use of multiple CPUs on a multi-processor computer. This obviously requires a compute with more than one CPU, but also a POSIX compatable thread library. Most systems, including GNU/Linux, normally have such a library installed, although a minimal setup of the operating system might well not have such a library present. You can if you wish use the GNU Pth library at http://www.gnu.org/software/pth/ In order to configure atlc as a multi-threaded program for use on multiple CPUs, use the --with-threads option to the configure script i.e. run

    configure --with-threads
    make
    make check
    make install
  2. Use of the processing power of a number of CPUs on a network of computers.The code for this is experimental and has been temporarily disabled, since it did not work properly. So only attempt to use it if you are willing to put some effort into the development. The distributed processing uses the Message Passing Interface (MPI) and requires the MPICH libraries from http://www-unix.mcs.anl.gov/mpi/mpich/indexold.htm In order to experiment with this, configure atlc with the --with-mpi option to the configure script i.e. run

    configure --with-mpi
    make
    make check
    make install
